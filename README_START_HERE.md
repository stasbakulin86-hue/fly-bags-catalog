# FLY-BAGS — готовый сайт-каталог для GitHub Pages

Это стартовая версия сайта-каталога для магазина чемоданов.

Внутри уже есть:

- главная страница;
- каталог товаров;
- фильтры по размеру, цене, цвету, материалу и наличию;
- карточки товаров;
- окно с характеристиками;
- мобильная версия;
- фирменные цвета FLY-BAGS: белый, синий, тёмно-синий, жёлтый;
- тестовый файл `products.json`;
- папка `images` с изображениями-заглушками;
- шаблон CSV для Airtable.

## Как проверить сайт на компьютере

Откройте файл `index.html` двойным кликом.

Важно: в некоторых браузерах товары из `products.json` могут не загрузиться при открытии файла напрямую с компьютера. На GitHub Pages всё будет работать нормально.

## Как загрузить на GitHub Pages

1. Зарегистрируйтесь или войдите на GitHub.
2. Создайте новый репозиторий, например `flybags-catalog`.
3. Выберите `Public`.
4. Загрузите все файлы и папки из этой папки в репозиторий.
5. Откройте `Settings` → `Pages`.
6. Выберите:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
7. Нажмите `Save`.
8. GitHub выдаст ссылку на сайт.

## Как менять товары вручную

Откройте файл `products.json` и добавьте/измените товары.

Главные поля:

- `id` — артикул;
- `title` — название;
- `category` — категория;
- `size` — размер;
- `price` — цена;
- `oldPrice` — старая цена, если нужна;
- `color` — цвет;
- `material` — материал;
- `image` — путь к фото;
- `status` — `show` или `hide`;
- `featured` — `true`, если товар должен быть в популярных.

## Как скрыть товар

В товаре замените:

```json
"status": "show"
```

на:

```json
"status": "hide"
```

## Как заменить изображения

1. Откройте папку `images`.
2. Загрузите туда свои фото, например `fb-m-blue.jpg`.
3. В `products.json` укажите путь:

```json
"image": "images/fb-m-blue.jpg"
```

## Как потом подключить Airtable

Эта версия пока работает от файла `products.json`.
Следующий этап — сделать автоматическую выгрузку:

Airtable → GitHub Actions → products.json → сайт.

Для этого понадобится:

- Airtable Base ID;
- Airtable Personal Access Token;
- GitHub Secrets;
- workflow-файл для GitHub Actions.

## Важное

GitHub не создаёт дизайн автоматически. Дизайн уже находится в файлах:

- `index.html` — структура сайта;
- `style.css` — внешний вид;
- `script.js` — фильтры и загрузка товаров.

## Промокоды

В этой версии добавлена функция промокода.

Файл с промокодами:

```text
promocodes.json
```

Тестовые промокоды уже добавлены:

- `FLY300` — скидка 300 ₽ на любой чемодан от 2 990 ₽;
- `FLY500` — скидка 500 ₽ на чемоданы M и L;
- `SET1000` — скидка 1 000 ₽ на наборы;
- `SALE10` — скидка 10% на товары от 4 000 ₽.

Как изменить промокод:

```json
{
  "code": "FLY500",
  "title": "Скидка 500 ₽ на чемоданы M и L",
  "type": "fixed",
  "value": 500,
  "sizes": ["M", "L"],
  "minPrice": 3500,
  "active": true
}
```

Что значат поля:

- `code` — сам промокод;
- `title` — описание для сайта;
- `type` — `fixed` для скидки в рублях или `percent` для скидки в процентах;
- `value` — размер скидки;
- `minPrice` — минимальная цена товара;
- `sizes` — размеры, на которые действует промокод;
- `categories` — категории, на которые действует промокод;
- `active` — `true`, если промокод включён, и `false`, если выключен.

Важно: сайт на GitHub Pages статичный, поэтому промокод используется как расчёт скидки на витрине. Окончательную скидку подтверждает продавец в магазине, Авито или переписке.
